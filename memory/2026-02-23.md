### Browser Configuration (Headless Mode)
**Goal:** Configure browser for headless operation on server without display

**Challenge:** Snap Chromium + AppArmor confinement prevents direct browser launch  
**Solution:** Configured attach-only mode + manual browser start

**Config changes:**
```json
{
  "browser": {
    "enabled": true,
    "headless": true,
    "noSandbox": true,
    "attachOnly": true,
    "defaultProfile": "openclaw"
  }
}
```

**Browser start command:**
```bash
chromium-browser --headless=new --no-sandbox --disable-gpu \
  --remote-debugging-port=18800 \
  --user-data-dir=$HOME/snap/chromium/common/openclaw-browser \
  about:blank &
```

**Result:** ✅ Successfully navigated to google.com and captured screenshot

**Key learnings:**
- Snap Chromium requires user-data-dir inside `~/snap/chromium/common/` for write access
- AppArmor restrictions prevent OpenClaw from launching browser directly
- Attach-only mode is the workaround for snap environments
- Could install Google Chrome .deb as alternative (no snap restrictions)
