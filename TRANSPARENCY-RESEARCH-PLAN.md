# Transparency Research Plan (v1)

## Objective
Define practical anti-black-box principles for AI-assisted work, then operationalize them across all Oscar outputs.

## Phase 1 — Research (start here)
Questions:
1. What transparency artifacts create real trust (not performative overhead)?
2. What minimum evidence is needed per output type?
3. How should confidence/uncertainty be expressed?
4. What level of traceability is useful day-to-day vs audit mode?
5. How to keep transparency lightweight enough to sustain?

Deliverable:
- `TRANSPARENCY-STANDARD.md` draft (principles + required fields)

## Phase 2 — Integration
- Add "How this was derived" section to templates
- Add `TRACEABILITY-INDEX.md` linking specs/scenarios/tests/decisions
- Add checklist gate in delivery loop

## Phase 3 — Validation
- Apply standard to 3 recent artifacts
- Measure usefulness + overhead
- Refine fields using 80/20 principle

## Success Criteria
- Chris can quickly answer:
  1) what was done,
  2) why,
  3) based on what evidence,
  4) how confident,
  5) what could still be wrong.
